# 业务规则手册

> 从需求文档提取的核心业务规则。

## 1. 角色与权限

### 1.1 用户角色

| 角色 | 说明 |
|------|------|
| **超级管理员** | 全权限，所有操作均可执行 |
| **档案管理员** | 档案管理、审批权限申请（目前仅一人待确认） |
| **普通查询员** | 查询/查看已有档案 |

### 1.2 权限体系

权限分两层：

| 层级 | 说明 |
|------|------|
| **企业级访问控制** | A 企业可查看 A 企业内的所有文件（大权限） |
| **文件级权限** | 单个文件内的细粒度权限：是否可查看、是否可下载（小权限） |

> ⚠️ 技术上是否可以精确控制查看和下载权限，待开发阶段验证。

### 1.3 权限申请流程

```
用户提交 → 档案管理员授权/驳回
```

审批时对接短信平台通知申请人。

## 2. 档案管理

### 2.1 档案状态

| 状态 | 说明 |
|------|------|
| **已归档** | 审批通过，显示推送功能按钮 |
| **未归档** | 审核中/待处理，不显示推送按钮 |

状态的唯一作用是控制是否显示推送功能按钮。

### 2.2 分类体系

两级分类（一级 → 二级），全局统一：

| 一级分类 | 二级分类 |
|----------|----------|
| 规划资料 | 立项备案、用地许可、总图、规划许可 |
| 建设资料 | 施工图审查、施工许可、其他 |

- 分类/标签体系全局统一（每个项目都一样）
- 管理员可随时新增修改，便于后续业务扩展

### 2.3 关联关系

档案与项目/企业的关联关系**通过上传操作建立**，系统不处理逆向关联。同一份文件如需出现在多个项目中，由用户在不同项目中分别上传。

## 3. 文件管理

### 3.1 文件格式

| 场景 | 规则 |
|------|------|
| **上传** | 格式不做限制 |
| **预览** | 支持 PDF、图片格式（政府端+企业端均可） |
| **下载** | 有权限即可下载，无需预览 |

### 3.2 历史版本

已删除的文件通过历史版本记录查看（保留删除前的权限配置），可预览/下载已删除的文件。

## 4. 推送功能

- 推送指**政府端各部门用户间**的档案推送
- 仅**已归档**档案可推送
- 仅**指定用户及超级管理员**有推送权限
- 推送时对接短信平台通知接收人
- 支持按新增人员推送，已推送人员不重复推送
- 推送后可查看推送名单

## 5. 企业端功能

- 企业端仅查看本企业名下所有项目档案（数据隔离）
- 企业端撤回简化：政府端审批完成前，企业可直接撤回（无需申请流程）
- 政府端审批**驳回即结束**，不循环
- 项目名称由政府端统一管理，企业端只能查看

## 6. 短信通知

涉及两个场景：

| 场景 | 通知对象 |
|------|----------|
| 推送档案时 | 通知接收人 |
| 审批操作时 | 通知申请人 |

已有短信平台接口文档，直接对接调用。

## 7. 日志

- 记录所有操作：登录、查询、上传、下载、修改、删除等
- 日志不可篡改
- 支持按用户名、姓名、部门、时间段筛选
- 支持日志导出
