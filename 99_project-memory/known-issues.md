## 🔵 第一阶段：原型系统

### 未修复

### 1. 预览 HTTPS 混合内容
- 当前 3 处预览 URL 使用 `http://183.215.41.66:8012`（明文 HTTP）
- 如果用 HTTPS 页面嵌入，会产生混合内容警告
- 演示阶段可接受，生产需配 SSL

### 2. 原型 Mock 数据不足
- 档案数据仅 140 条，未覆盖所有业务场景
- 企业端 Mock 数据较简略
- 推送、权限申请等弹窗为模拟，无实际后端交互

### 3. KKFileView 内存占用
- KKFileView 容器占用约 814MB 内存
- 演示服务器内存充裕（31G），但生产需注意资源规划
- 长时间运行建议配置 JVM 参数限制内存

### 4. 预览地址硬编码
- 原型中 3 处预览 URL 硬编码为 `183.215.41.66` IP 地址
- 如果更换服务器或域名，需手动修改
- 后续正式开发应改为配置

## 已修复

### ~~预览 URL 指向本地地址~~
- **症状**：点击在线预览时跳转 `localhost:8012` 导致浏览器端访问失败
- **原因**：原型中原先使用 `host.docker.internal:9000` 和 `localhost:8012` 本地地址
- **修复**：改为外网 IP `183.215.41.66` 对应端口（3处全部更新）
- **日期**：2026-06-16

### ~~预览连接被拒绝~~
- **症状**：KKFileView 报"Connection refused"
- **原因**：Python 文件服务器未运行；文件路径缺少 `demo/` 前缀
- **修复**：启动端口 9000 文件服务器；修正路径加 `demo/`
- **日期**：2026-06-10

### ~~标签文字颜色不可见~~
- **症状**：项目档案一览弹窗中"部分归档"标签文字和背景同色
- **原因**：CSS 优先级冲突，`.tag-warning` 被后续规则覆盖
- **修复**：使用独立类名 `.pj-partial` 和 `.pj-archived`
- **日期**：2026-06-14

### ~~弹窗标题栏与内容区间隙过大~~
- **症状**：标题栏底部与"企业名称"之间有 16px 空白
- **原因**：标题栏 `padding-bottom: 16px` + body `padding-top: 16px`
- **修复**：`.pj-modal` 类控制标题栏 `padding-bottom: 0`、body `padding-top: 0`
- **日期**：2026-06-14
