> 基于合同 Article 3 及 XMind 合并整理（已与客户确认）

## 一、政府端（PC 端）

### 1.1 首页

| 编号  | 功能模块 | 功能点   | 说明                                                                                                                                                                                                                                                                                                                                                                                                                                 |
| ----- | -------- | -------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------ |
| 1.1.1 | 首页     | 推送消息 | 展示系统推送消息，**点击后消除提醒**。消息来源：档案审核结果通知、权限审批通知、档案推送通知、短信通知。 |
| 1.1.2 | 首页     | 待办消息 | 展示待办事项，点击进入待办事项，**办理完后消除提醒** |
| 1.1.3 | 首页     | 数据总览 | 通过数据可视化方式展示系统运行数据，**每1小时刷新一次**：<br />1、平台存储档案总数、已归档档案总数、未归档档案总数、累计查询次数总数；<br />2、各档案分类的已归档档案总数，各项目的已归档档案总数，各企业上传的已归档档案总数；<br />3、分企业展示上传档案总数，分月度展示上传数量；<br />4、当前系统在线人数；按时间线倒序展示最近档案上传和审核事件。 |
| 1.1.4 | 首页     | 全文检索 | 快捷入口，输入关键词跳转到**档案查询**结果页 |
| 1.1.5 | 首页     | 精准检索 | 快捷入口，点击打开弹窗输入精准匹配条件，跳转到**档案查询**结果页 |

### 1.2 档案管理

#### 1.2.1 档案查询

| 编号     | 功能点         | 说明                                                                                                                                                                 |
| -------- | -------------- | -------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| 1.2.1.1  | 多条件组合查询 | 支持按**企业名称、项目名称、文件名称**等多条件组合查询档案                                                                                                     |
| 1.2.1.2  | 全文检索       | 具备全文检索功能，可检索档案文件的**正文内容**和**标题**                                                                                                                     |
| 1.2.1.3  | 标签筛选搜索   | 支持筛选企业、项目、分类后，再通过**标签**进行搜索匹配。分类/标签全局统一，可随时配置新增                                                                                                         |
| 1.2.1.4  | 推送归档档案   | 政府端各部门用户间推送档案。手动点击推送，选择接收人员主动推送。推送后状态更新，可查看推送名单，支持**新增人员推送**，已推送人员不重复推送。推送功能仅**已归档档案**可操作，仅**指定用户及超级管理员**有权限。推送（及审批）时对接短信平台发送短信通知。 |
| 1.2.1.5  | 在线预览/下载  | 上传文件格式不做限制。预览支持 **PDF、图片**格式；压缩包文件支持下载；其他格式文件有权限即可下载，无需预览。无权限则无法预览和下载。 |
| 1.2.1.6  | 历史版本记录   | 查看档案时，通过历史版本记录查看已删除的文件和时间，可预览/下载                                                                                                      |
| 1.2.1.7  | 已删除文档权限 | 已删除文档的查看/下载权限与**删除前配置保持一致**，可通过权限申请审批获得                                                                                      |
| 1.2.1.8  | 企业权限限制   | 企业权限分两层：<br />1、**大权限**：A 企业可查看 A 企业内的所有文件（企业级访问控制），限制后无法进入档案详情，但可搜索到对应档案；<br />2、**小权限**：单个文件内的细粒度权限：是否可查看、是否可下载。<br />如需访问需通过**权限申请审批** |
| 1.2.1.9  | 批量权限申请   | 支持在档案详情页通过申请权限，**批量**对当前项目下所有无权限档案申请访问，也可**单独逐个**提交申请                                                       |
| 1.2.1.10 | 操作日志记录   | 档案查询、推送、预览、下载等操作均记录日志                                                                                                                           |

#### 1.2.2 档案管理

| 编号    | 功能点            | 说明                                                                                                                                                                                                                 |
| ------- | ----------------- | -------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| 1.2.2.1 | 企业名称选择/输入 | 企业名称支持文本框**模糊匹配**，选择企业库内已有企业；若无完全匹配，可手动输入，但无法匹配企业账号                                                                                                             |
| 1.2.2.2 | 项目名称查重      | 新建项目名称时需查重项目名称，存在相同名称时**不允许新建**，提示进入对应项目更新。项目名称由**政府端统一管理**，企业端只能查看。                                                               |
| 1.2.2.3 | 档案信息录入      | 需选择或输入：企业全称、项目名称全称、档案**一级分类和关联二级分类**，支持**上传多个文件**                                                                                                               |
| 1.2.2.4 | 单文件权限设置    | 上传文件后，可针对**单个文件**设置查看/下载权限                                                                                                                                                                     |
| 1.2.2.5 | 档案上传          | 政府端用户创建并上传档案。**上传文件类型不做限制**。上传时需选择档案关联的企业名称、项目名称、档案分类等。同一份文件如需出现在多个项目中，由用户在不同项目中分别上传（系统不做自动关联）。 |

| 1.3.10 | 档案标签管理 | 增删改查     | 对标签进行增删改查                                                                                     |

## 四、关键业务规则

### 4.1 档案状态

| 状态 | 说明 |
|------|------|
| **已归档** | 审批通过，显示推送功能按钮 |
| **未归档** | 审核中/待处理，不显示推送功能按钮 |

### 4.2 分类与标签

- 每个项目的类别/分类/标签体系**全局统一**
- 可通过后台随时新增修改，便于后续业务扩展

### 4.3 关联关系

档案与项目/企业的关联关系**通过上传操作建立**，系统不处理逆向关联。同一份文件如需出现在多个项目中，由用户分别上传。

### 4.4 推送范围

推送是指**政府端各部门用户间**的档案推送（如行政审批局推送至产业发展局）。推送（及审批）时对接短信平台发送通知。

## 五、讨论确认记录

**已确认：**
- 首页-推送消息来源：档案审核结果通知、权限审批通知、档案推送通知
- 首页-数据总览刷新频率：每1小时刷新一次
- 首页-全文检索/精准检索：首页为快捷入口，跳转到档案查询结果页
- 档案查询-全文检索范围：包括正文内容和标题
- 一份档案**不会关联多家企业**，可通过不同项目分别上传
- 权限申请审批流程：**提交 → 档案管理员授权/驳回**
- 企业权限分两层：企业级访问控制 + 文件级权限（查看/下载）
- 用户角色：**超级管理员、档案管理员、普通查询员**（档案管理员仅一人待确认）
- 政府端/企业端均需在线预览，上传格式不限，预览支持 PDF/图片
- 企业端撤回简化：政府端审批完成前可直接撤回，驳回即结束不循环
- 档案分类和标签配置可随时新增修改
- 新增短信通知功能（推送 + 审批场景），对接已有短信平台
- 档案状态：仅**已归档/未归档**两种
- 项目名称由政府端统一管理
- 推送范围限定为政府端各部门用户间
